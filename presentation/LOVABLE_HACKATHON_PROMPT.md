# Prompt для Lovable

Ниже prompt, который нужно вставить в Lovable, чтобы собрать slide-based presentation website для скриншотов в финальную презентацию.

```text
Сделай презентационный сайт для финальной защиты хакатона перед жюри.

Важно:
Это не обычный landing page.
Это должен быть сайт в формате pitch deck, из которого я сделаю screenshot-слайды для Demo Day.
У меня будет живое демо продукта на ноутбуке, поэтому этот сайт должен усиливать pitch, а не пересказывать каждое действие в интерфейсе.

Контекст задачи:
Проект участвует в хакатоне по внедрению AI в HR-процессы управления эффективностью персонала.
По ТЗ важно показать:
- AI-оценку качества и релевантности целей
- генерацию 3–5 целей на основе ВНД, KPI и целей руководителя
- привязку каждой цели к источнику из ВНД
- контроль набора целей: количество, вес, дублирование, тип цели
- dashboard зрелости целеполагания по подразделениям
- explainable hybrid AI architecture, а не просто LLM demo

Название проекта:
KMG HR AI Command Center

Ключевой тезис:
Это не просто SMART-checker.
Это AI-слой над Performance Management для оценки качества целей, стратегической связки и генерации сильных формулировок.

Формат сайта:
- one-page slide-based presentation website
- fullscreen sections
- каждый экран = отдельный слайд
- 16:9 screenshot-friendly composition
- vertical snap scroll
- очень чистая композиция
- минимум текста
- сильная типографика
- каждый экран должен выглядеть как готовый слайд для жюри

Стиль:
- premium corporate tech
- deep navy / graphite / soft cyan / mint accents
- без фиолетового
- без generic startup landing page
- без длинных абзацев
- без мелкого текста
- без однотипных карточек на каждом экране
- ощущение дорогой, уверенной и explainable AI platform
- крупные заголовки
- короткие тезисы
- slide number в каждом экране

Сделай 7 слайдов:

Slide 1. Hero
Заголовок:
KMG HR AI Command Center

Подзаголовок:
AI-слой над Performance Management для оценки качества целей, стратегической связки и генерации сильных формулировок.

Короткий тезис:
Не просто SMART-checker, а command center для качества целеполагания.

Добавь 3 коротких chips:
- Оценка целей
- Генерация 3–5 целей
- Dashboard зрелости

Slide 2. Проблема
Заголовок:
Почему обычный goal setting не работает

Покажи 4 business pain:
- SMART не гарантирует связь цели с бизнесом
- цели часто не связаны с KPI, ВНД и целями руководителя
- в наборе появляются activity-based цели, дубли и перекос по весам
- review становится ручным и формальным

Сделай этот экран как business pain slide, а не как текстовую страницу.

Slide 3. Решение
Заголовок:
Что делает KMG HR AI Command Center

Покажи 4 capability blocks:
- Goal evaluation
- Goal generation
- Set governance
- Team radar

Краткие русские подписи:
- SMART + релевантность роли + стратегическая связка
- 3–5 целей на основе ВНД, KPI и каскада руководителя
- контроль количества, веса, дублирования и типа цели
- индекс зрелости целеполагания по подразделениям

На этом слайде важно визуально показать, что это не одна фича, а целостная система.

Slide 4. Соответствие ТЗ
Заголовок:
Как решение закрывает MVP и критерии хакатона

Покажи 6 proof blocks:
- AI-оценка одной цели
- Переформулировка слабой цели
- Генерация 3–5 целей
- Источник + цитата из ВНД
- Контроль набора целей
- Dashboard по подразделениям

Сделай этот слайд очень убедительным для жюри.
Он должен показывать, что решение собрано по ТЗ, а не просто как AI showcase.

Slide 5. Credibility / Architecture
Заголовок:
Почему это не просто LLM demo

Покажи 4 блока:
- RAG over VND and strategy
- business context: роль, подразделение, KPI, цели руководителя
- deterministic checks: SMART, тип цели, дублирование, count/weight alerts
- optional LLM for wording and generation

Нужен сильный credibility slide.
Без перегрузки техническими деталями.
Без схемы уровня enterprise architecture.

Slide 6. Бизнес-эффект
Заголовок:
Какой эффект получает компания

Покажи 4 outcome blocks:
- меньше формальных целей-активностей
- выше стратегическая связка
- быстрее review для руководителя и HR
- выше зрелость Performance Management процесса

Сделай этот экран как value slide для бизнеса и жюри.

Slide 7. Closing
Заголовок:
От формального SMART — к управляемому Performance Management

Подзаголовок:
KMG HR AI Command Center помогает перейти от формальной постановки целей к стратегически связанным, объяснимым и измеримым целям.

Footer:
GitHub: github.com/temirrrr/hr-ai-app

Дополнительные требования:
- не делай demo-flow slide
- не делай длинные paragraphs
- чередуй композиции: hero / pain / capability / proof / credibility / value / closing
- сделай всё screenshot-friendly
- дизайн должен выглядеть как финальная защита хакатона, а не маркетинговый лендинг
- выбери убеждающий и строгий visual language

Если нужно выбирать между “красиво” и “убедительно для жюри”, выбирай убедительно для жюри.
```
